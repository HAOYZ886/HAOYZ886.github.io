前言

这篇文章很意识流，想到什么就说什么，但能通过此学习时钟、电源和 Pinctrl 。

目的

MIPI-DSI 屏和 LVDS 屏的 LOGO 延续从 U-Boot 阶段延续到内核阶段，直到 Weston 桌面启动。

问题

在 U-Boot 期间，在 MIPI-DSI 屏和 LVDS 屏生成 LOGO 。从 U-Boot 切换到 Kernel 后，屏幕先出现花屏，然后被像素图像刷掉，再接着背光被关掉。而且出现花屏的情况仅在内存容量为 2G 的 IMX8MP 出现。

理解

文中的保序是指 CPU 之间的保序，或 CPU 与 DMA 之间的保序，保序的操作就是禁止乱序执行和内存重排序。
这个保序像 CPU 之间的同步操作，如：

当 CPU0 执行到 flag = 1; 后，CPU1 才能执行 if (flag == 1) 。比如，不能把 flag = 1; 被重排到 if (flag == 1) 的后面；变量 a 的加载不能在 if (flag == 1) 之前（分支预测搞的鬼）。
综上所知，在内存屏障之前的指令要完成，然后才能执行屏障后面的指令。
