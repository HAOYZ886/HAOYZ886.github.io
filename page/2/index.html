内存屏障学习笔记

Posted on 2025-06-27 Edited on 2025-06-30 In 原理及实现 Word count in article: 1.6k Reading time ≈ 6 mins.

内存屏障出现的原因

CPU流水线

指令重排序

特性

  • 特点：是指通过编译器 -O2 或 -O3 优化，使得让访问内存（变量）的次数减少，把变量值存储在寄存器内，读写操作在寄存器进行。
  • 缺点：对于 SMP 系统，在多核共享同一个变量的情况下，如果变量值一直存储在寄存器内，导致内存一致性无法得到保证。
  • 解决：用关键字 volatile 修饰变量；或在关键代码处（需要查看经过优化后的汇编代码）添加 asm volatile("" ::: "memory") ，使得编译器不对该处代码优化。
  • 新的问题：这样已确保变量值都写到内存，但无法保证内存访问的顺序一致性。

Linux - 烧录安路FPGA固件驱动

Posted on 2025-06-20 In 原理及实现 Word count in article: 1.5k Reading time ≈ 5 mins.

硬件说明

FPGA型号：EG4X

该功能是基于安路的从动串行加载模式，FPGA 作为从器件，由 CPU 控制加载位流。可以使用 .bin 或 .rbf 文件配置 FPGA 。

使用 .bin 文件，则从每个字节高位开始往外串行送数据。若使用 .rbf 文件，则从每个字节低位开始往外串行送数据。CPU 通过 CCLK、DIN 信号使用串行方式将数据写入 FPGA 。芯片在每个 CCLK 的上升沿接收数据，数据接收完成后，DONE 拉高表示配置完成，如果配置过程中出错，FPGA 会将 INIT_B 信号拉低。
