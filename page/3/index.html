Linux - Coredump调试

Posted on 2025-04-29 In Skill Word count in article: 2.2k Reading time ≈ 8 mins.

前言

Coredump 是指在程序异常终止时，操作系统将程序的内存映像保存到磁盘上的一种机制。

在 Linux 系统中，core dump 提供了一种调试程序错误的重要方式，它记录了程序在崩溃时的内存状态，可以帮助开发人员快速定位问题。当程序因为某种原因（如段错误、非法指令等）异常终止时，Linux 系统会尝试将程序在内存中的映像、程序计数器、寄存器状态等信息写入到一个名为 core 的文件中，这个文件就是所谓的 core dump。

Linux - 深入分析ww-mutex（内核版本 6.1.55）

Posted on 2025-04-24 Edited on 2025-05-04 In 原理及实现 Word count in article: 10k Reading time ≈ 37 mins.

参考

Documentation/locking/ww-mutex-design.rst

发现

在阅读 DRM 源代码时，发现使用了一个特别的锁，名为 ww-mutex ，该锁是 mutex 的“升级版”。ww-mutex 的全称是 wound-wait mutex。
在 DRM 中，如 drm_helper_probe_single_connector_modes() 和 ATOMIC COMMIT 操作等，调用 drm_modeset_lock() 和 drm_modeset_drop_locks() 去获取和释放 drm_modeset_lock 。drm_modeset_lock 的基础对象就是 ww-mutex 。
