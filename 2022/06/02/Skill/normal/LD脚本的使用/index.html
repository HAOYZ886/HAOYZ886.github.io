Posted on 2022-06-02 In Skill Word count in article: 1.6k Reading time ≈ 6 mins.

链接脚本是什么

了解什么是链接脚本前，需要知道编译过程
以 C 语言为例，已知从源码文件编译为可执行文件分为4个步骤：

  1. 预处理：尝试打开使用 include 引用的“.h”头文件，处理宏（如展开#define、判断条件#ifdef等）、处理声明函数（目的是识别返回值类型、形参数量来确认在栈中所需的空间）、处理声明全局变量等。
  2. 汇编：把 C 源码汇编为 AT&T 格式的汇编源码
  3. 编译：从汇编源码编译为目标文件（待链接文件），目标文件的头部存在 elf 头，elf 头存在一个节头表“section list”，表内每一行存储这每个节的基本信息，如：名称、类型、起始虚拟地址、文件内偏移量、大小、标签以及对齐方式等。
  4. 链接：把目标文件链接为可执行文件，可执行文件的 elf 存在两个表，一是节表“section list”，二是程序头表”segment list“。段是由可执行文件的节映射而来，段也是分类型的，如代码段、数据段等。每个段的信息被记录在程序头表内，如：文件内偏移量、大小、标签、和起始虚拟地址等。个人认为，程序头表更多的是告诉操作系统（或加载程序的任务线程）如何去加载程序。节表表示源码编译后的整体布局以便于链接，再由节映射段，可使用命令查看节与段的映射关系：readelf -e <file>。

    用命令查看 elf 头：readelf -e <file>

现在开始了解链接脚本，简单点地说，链接脚本是：告知 ld 程序如何把一至多个待链接文件链接为一个可执行文件，主要的操作对象是待连接文件的“节”，输出对象是可执行文件的“节”和“段”。最主要的操作有：设置一个程序的入口地址、如何把多个类型的节组合成一个节、把一个节映射段后加载到内存的哪里以及哪些节是不值得被处理的，等等。

关键字ENTRY()的作用是指定程序的入口地址（虚拟内存地址），symbol是一个符号/标签。看 kernel.ld 和 grub_head.S 的一个节段：

可以发现，在链接脚本 kernel.ld，start被设为程序的入口地址。在汇编源码文件 grub_head.S，标签start被设为全局，同时也表示了一个一段代码的起始虚拟地址。所以当程序开始时，CPU 的 EIP 指向start作为程序的入口地址开始读取执行指令。
因为 boot/grub_head.S 编译为目标文件后，通过链接脚本 kernel.ld 被链接在节 .init.text，.init.text 的起始虚拟地址为 0x100000，所以 start 所指向的地址是 0x100000。
