bochs指令集

Posted on 2022-12-30 In Skill Word count in article: 1.1k Reading time ≈ 4 mins.

以下是本人经常使用到的 bochs 调试指令，记录一下。

Breakpoint

虚拟地址断点指令

vb|vbreak 段:偏移

线性地址断点指令

lb|lbreak 线性地址

物理地址断点指令

b|break 物理地址

指令数打断点指令

sb delta

再执行delta条指令程序就中断

sba time

CPU从运行开始，执行time条指令时中断，从0开始的指令数

读断点指令

watch r 物理地址

设置读断点，如物理地址有读操作就停止执行

写断点指令

watch w 物理地址

设置写断点，如物理地址写操作就停止执行

显示所有读写断点指令

watch

清除所有断点指令

unwatch

清除在此地址上的读写断点指令

unwatch 物理地址

显示所有断点信息指令

blist

禁用/启用断点指令

bpd|bpe n

bpd是禁用，bpe是启用，n是断点号，可用blist查询获取断点号

删除某断点指令

d|del n

n是断点号

debugger

退出调试命令

q | quit | exit

set命令集

设置通用/段寄存器的值

set reg = val

设置每次停止执行后，是否反汇编指令

set u on|off

show命令集

CPU变更模式时提示

show mode

发生函数调用（调用过程）时提示

show call

中断发生时提示，显示三种类型的中断，软中断、硬件中断、iret

show int

软中断发生时提示

show softint

硬件中断发生时提示

show extint

显示iret指令有关的信息

show iret

每次执行一条指令，是否将这条指令反汇编并显示到控制台

trace on|off

根据机器代码生成可读的汇编代码指令

u /num start end

execution

继续执行指令

continue|c

单步/n步执行指令

step|s count

执行count条指令，不使用参数则默认为1

执行一条指令

p|n|next

若执行的指令是函数调用（调用过程），会把整个函数当作整体来执行

information

查看指令的描述指令

help+命令

显示通用寄存器内容指令

register|r

执行循环指令

n

显示线性地址的内容指令

x /nuf 线性地址

n表示显示单元数（默认为1），u表示显示单元大小（b 1字节；h 2字节；w 4字节；g 8字节），f表示使用哪种进制表示内存单元（默认十六进制）

显示物理地址的内容指令

xp /nuf 物理地址

n表示显示单元数（默认为1），u表示显示单元大小（b 1字节；h 2字节；w 4字节；g 8字节），f表示使用哪种进制表示内存单元（默认十六进制）

设置连续的相同的值指令

setpmem 物理地址 size val

设置以物理地址为起始，连续size个字节的内容为val。PS：size最多只能设置4字节，超过会报错

显示通用寄存器、eflags、eip的值指令

r|reg

查看栈内的数据指令

print-stack num

num用于指定显示多少数据。不使用，则默认显示16个字
PS：输出的栈内容是栈顶在上，低地址在上，高地址在下。

info指令集

查看CPU所有的寄存器的值，包括不可见寄存器

info CPU

显示fpu状态

info fpu

查看标志位

info eflags

标志名称大写表示标志为1，小写为0

查看当前任务的页表信息

info tab

显示线性地址和物理地址（页）的全部映射关系

查看IDT的所有内容

info idt

显示内容前，ldtr寄存器先获取IDT的物理地址和表界限

查看GDT的所有内容

info gdt num

如果加了num参数，则显示第num条描述符。
PS：显示内容前，gdtr寄存器先获取GDT的物理地址和表界限

查看LDT的所有内容

info ldt

PS：显示内容前，ldtr寄存器先获取到LDT的基地址和段信息

查看TSS的内容

info tss

PS：显示内容前，tr寄存器先获取到TSS的基地址和段信息

显示中断向量表

info ivt

如果加了num参数，则显示第num条中断向量。

显示段寄存器内容指令

sreg

显示控制寄存器指令

creg

显示调试寄存器指令

dreg

显示线性地址到物理地址间的映射指令

page line_addr
