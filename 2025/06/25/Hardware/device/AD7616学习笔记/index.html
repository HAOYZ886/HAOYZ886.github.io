AD7616学习笔记

Posted on 2025-06-25 In Hardware Word count in article: 3.5k Reading time ≈ 13 mins.

前言

本文注重讲解 AD7616 的：

  • 配置方式为：软件模式
  • 传输方式为：并行模式
  • 读采样值的方式为：并行模式 + Sequencer + Burst 模式

参考资料

AD7616.pdf

概述

AD7616是一个16位DAS，支持16通道双同步采样。AD7616采用单个5v电源，可容纳±10v、±5v和±2.5 V真双极输入信号，同时以每通道对高达1 MSPS的吞吐量采样，信噪比为90.5 dB。片上过采样模式（92 dB，过采样比（OSR）为2）可以实现更高的信噪比性能。

输入钳位保护电路可以承受高达 ±21 V。AD7616的模拟输入阻抗为1 MΩ，与采样频率无关。单电源操作、片上滤波和高输入阻抗消除了对驱动运放和外部双极电源的需求。

该器件包含模拟输入箝位保护、双16位电荷再分配SAR模数转换器（ADC）、灵活数字滤波器、2.5 V参考和参考缓冲器以及高速串行和并行接口。

功能简述

功能解析

以下是它的功能视图：

见下图的红色框，引脚 V0A ~ V7A 和 V0B ~ V7B 是采集电压通道。

见下图的黄色框，其中，9:1 MUX的 Input 端包含V0A ~ V7A 或 V0B ~ V7B ，还有来自 2:1 MUX的输出端。可通过 CONTROL INPUTS（硬件模式，见灰色框），也可通过操作 AD7616 的寄存器（软件模式），选择采集哪些电压通道。而且在软件模式中，还能指定读取采样通道的顺序。也可以选择诊断通道（Vcc 和 ALDO），检测 AD7617 内部电压是否运行正常。

见上图的绿色框，会把输入端的电压通过16-BIT SAR模数转换器，转换为相关模数数值。再然后把该数值通过并行模式，从引脚<font style="color:rgb(42, 43, 46);">DB0~DB15</font>输出，见蓝色边框。

已知软件模式需要访问寄存器才能配置，如何访问寄存器？

在并行模式下，是通过<font style="color:rgb(42, 43, 46);">DB0~DB15</font>引脚指定寄存器地址，并把对应数值写入 AD7616 ，也可以通过<font style="color:rgb(42, 43, 46);">DB0~DB15</font>引脚指定寄存器地址，从 AD7616 读出寄存器的值。

那么怎么配置读操作和写操作？

在并行模式中，引脚<font style="color:rgb(42, 43, 46);">BURST</font>被作为写操作引脚，该引脚被拉低时表示开启写操作。引脚<font style="color:rgb(42, 43, 46);">SCLK</font>（上图中未被标出，给串行模式作时钟引脚）被作为读操作引脚，该引脚被拉低时表示开启读操作。

AD7616的硬件应用

指定 AD7616 的配置方式为软件模式，传输方式为并行模式，则 AD7616 本身的配置为：

  1. 配置SER/PAR引脚，该引脚被置位表示使用串行模式，被清除表示使用并行模式，故把该位清除。
  2. 把上图灰色框中的除了RESET、BURST、BUSY和CONVST之外的引脚用 GND 拉低，因为它们是用在硬件模式的。
  3. 在并行模式下，引脚OS2 TO OS0、SDOx/SDI 和SER1W都是作为引脚DB15 TO DB0。
  4. 另外，我们这边把引脚用3.3V电压REFSEL拉高，表示使用内部参考电压，把引脚VDRIVER也用3.3V拉高。所以见紫色框，把除REFSEL和VDRIVER之外的引脚，都直接接上 GND 。

AD7616寄存器说明

注意，仅有在软件模式下才能使用寄存器来配置 AD7616 ，硬件模式只能操作上图灰色框中的引脚来配置 AD7616 。

寄存器合集

以下是 AD7616 可操控的寄存组：

寄存器寻址

使用以下方式指定寄存器地址、读写操作和数据（仅限写操作）。

  • 位15是指定是要读出寄存器的值（把该位清除），还是把值写入寄存器（把该位置位）。注意不要和 AD7616 并行模式下的引脚 WR和RD混淆，这些引脚是用于指定传输数据方向，而位15的 W/R是用于指定读写寄存器。
  • 位 9~14 是指定寄存器地址去访问
  • 位 0~8 存储寄存器的数值，但仅限写寄存器操作，即位 15 被置位。

配置寄存器（CONFIGURATION REGISTER）

该寄存器可配置 sequencer 、Burst 模式、oversampling, 和 CRC 。

该寄存器在 T536 中仅配置为 sequencer 和 Burst 模式。

见以下定义：

  • 位 9~15 的作用是和《寄存器寻址》所讲的一致。
  • 位 5 SEQEN 被置位时，使能 sequencer 。使能后，序列栈寄存器组可被使用。
  • 位 6 BURSREN 被置位时，使能 Burst 模式。使能后，如该模式和 sequencer 配合，则仅需发出一次 CONVST 信号，就能一次性发送所有在序列栈寄存器组内指定的采集通道内的数模数值。
  • 其它位在 T536 暂未用到，忽略。详细见 AD7616 规格书的第 42 页。

那么这个寄存器组该如何正确使用？

虽然说该寄存器组叫序列栈寄存器组，但它和栈的特性无一点关系。你仅需顺着寄存器地址，去指定要读取通道的顺序。比如说，我要按顺序读取 (V0A和V0B) ~ (V7A和V7B) 通道的电压值，则可以这样配置：

寄存器地址 值（位0~7） 终止位（位8） 最终值（位15置位）
0x20 0x0000 0 0xC000
0x21 0x0011 0 0xC211
0x22 0x0022 0 0xC422
0x23 0x0033 0 0xC633
0x24 0x0044 0 0xC844
0x25 0x0055 0 0xCA55
0x26 0x0066 0 0xCC66
0x27 0x0077 1 0xCF77

其中终止位对应的是位 SSREN ，表示在此寄存器终止往下遍历，返回到起始处。

当你根据以上列表配置好寄存器后，AD7616 即可通过此配置，按顺序把 (V0A和V0B) ~ (V7A和V7B) 通道的电压值发出。发出完成后，会再次按之前配置的顺序发出，以此反复。

复位操作

复位操作分为部分复位和全复位，其中，部分复位是需要拉低 40~500 纳秒，如果超过 500 纳秒，则表示全复位，但全复位至少要拉低 1.2 微秒。

另外，部分复位和全复位的区别是，部分复位仅重置以下模块：

  • Sequencer
  • Digital filter
  • SPI
  • Both SAR ADCs

而且在软件模式下，部分复位不会把寄存器重置。

通常来说，都是用全复位来重置 AD7616 。
