DRAM的基础概念

DRAM子系统

DRAM子系统是我们整个嵌入式系统的核心，由多个硬件和逻辑组件协同工作，实现数据的存储、访问和管理。

DRAM子系统组成：主控SOC、传输链路、DRAM颗粒、电源。见示例图：

见以下各组件的解释：

  • 主控SOC：SOC 内的 DRAM 子系统包含控制器和 PHY ，通常称为 DDR IP 。
  • 传输链路：通常包含基板、PCB 走线等，影响信号质量，与子系统的极限性能有直接关系，设计与制造均需要关注
  • DRAM颗粒：占子系统成本的主要部分，其 IO 性能以及存储单元稳定性是主要关注点
  • 电源：容易忽略部分，主要关注路径直流压降以及应用端的纹波系数

见以下各条链路的解释：

  • CK：时钟信号，由 SOC 发送到 DRAM 。它是差分信号，包含 CK_t 和 CK_c 。用于提供时序同步基准，DRAM的所有操作（命令、地址、数据传输）均以CK的**边沿（上升沿/下降沿）**为触发条件。数据传输时，DQ和DQS的时序也需与CK对齐。
  • CA：命令/地址总线，由 SOC 发送到 DRAM 。传输内存操作指令和地址信息，是 SOC 控制 DRAM 的核心信号。命令信号 CKE、RST_N、CS#、ODT、RAS、CAS、WE、ACT 和地址信号** A0-A17、BG0-BG1、BA0-BA2 **等，用于发送DRAM的相关控制命令和寻址，依据JEDEC协议规定。
    • 命令：如激活（Activate）（打开某行）、读（Read）、写（Write）、预充电（Precharge）等。
    • 地址：行地址（Row Address）、列地址（Column Address）、Bank地址（内存分区标识）。
  • DQS ：数据选通信号，双向。解决数据与参考时钟的相位偏差问题，确保数据采样准确。
    • 写操作 wDQS（SOC→DRAM）：SOC 生成 DQS ，与 DQ 数据同步。DRAM 在 DQS 边沿采样 DQ 上的数据（例如：DQS 上升沿时捕获 DQ 值）。
    • 读操作 rDQS（DRAM→SOC）：DRAM 生成 DQS ，SOC 以 DQS 边沿为基准捕获 DQ 数据（避免因 PCB 走线延迟导致数据错位）。
  • DQ：数据线，双向（写时SOC→DRAM，读时DRAM→SOC）。实际传输数据的物理通道，直接决定内存带宽。
    • 写操作 **wDQ **：SOC 将 CPU 需要存储的数据通过 DQ 发送到 DRAM 。
    • 读操作 **rDQ **：DRAM 将存储的数据通过 DQ 返回给 SOC（最终送至 CPU ）。
    • 位宽影响：DQ 的位宽（如64位）越大，单次传输数据量越高（例如：64bit×800MHz=5.12GB/s带宽）。
  • VCC/VREF：电源/参考电压，电源管理 → DRAM。
    • VCC：DRAM的工作电源电压，为芯片内部电路（如存储单元、控制逻辑）提供能量。见以下供电链路：
      • VCC18-DRAM：用于给内部PHY等供电
      • VDD-DRAM：给数字电路供电，一般与VDD-SYS合并
      • VCC-DRAM/VCC-DRAML：IO供电，对应颗粒VDDQ
      • VDD18-LPDDR：颗粒端VDD1供电
      • VPP25-DRAM：DDR4专有，行激活电压
    • VREF：参考电压，用于 DQ 信号的电平判决。
      • 例如：DRAM 和 SOC 根据 VREF 设定阈值（如 VRE F= 0.5V ），判断 DQ 是逻辑1（>VREF）还是逻辑0（<VREF）。
      • 若 VREF 不稳定，会导致数据误判（如噪声干扰时本应是1却判为0）。

DDR IP

DDR IP（知识产权核）是集成电路设计中用于连接处理器与动态随机存储器（DRDRAM）的关键模块，涵盖控制器（Controller）和物理层接口（PHY）。

DDR IP 的组成：Host Controller 、PHY 。见以下示例图：

见以下各组件的解释：

  • 控制器：整个子系统的性能、功能、稳定性的根本，所有控制/操作命令的发起者
  • PHY：DDR PHY是连接DDR颗粒和DDR Controller的桥梁。负责把DDR Controller发过来的数据转换成符合DDR协议的信号，并发送到DDR颗粒；也负责把DRAM发送过来的数据转换成符合DFI协议的信号并发送给内存控制器。

传输链路设计

DRAM子系统链路设计，内容覆盖高速信号设计的各个关键点，主要包括但不限于以下部分：

  • DIE内设计
  • 基板设计
  • 封装ball设计
  • 原理图设计
  • PCB设计

以上设计是构建高性能内存架构的核心环节，需兼顾 信号完整性（SI）、电源完整性（PI）、时序收敛 和 成本控制，所以成熟的DRAM模板不可轻易修改。

DRAM颗粒(Die)

DRAM颗粒是物理 DRAM 芯片，比如内存条上的黑色小方块或核心板上的黑色小方块。

DRAM 的内部结构主要由一系列的物理结构和逻辑单元组成。

DRAM 颗粒的组成：Rank、Channel、Bank、Row 和 Column 。

一个 DRAM 颗粒有若干个 Bank ，Bank 就是一个个独立的内存阵列，用于提高并行访问效率：

需要注意的是，同一个 Bank 不能同时 Active 两个 Row 。另外，在访问当前的 Bank 时，可以提前对即将访问的下一个 Bank 进行预充电（Precharge），目的是减少等待时间，加快访问效率。

**Channel **是 CPU 内存控制器与 DRAM 颗粒之间的物理接口（比如说 双通道 和 四通道），每个 Channel 拥有一组独立的数据线、地址线和控制信号（如时钟、片选信号），多个 Channel 可并行传输数据，对带宽有提升效果。

位宽就是 Bit 的长度。全志 SoC 的 DRAM 位宽一般为 32 Bit 。DDR3/DDR4 的单颗粒一般为 16bit ，LPDDR4/LPDDR4X 单颗粒为 32bit 。Channel 也有位宽的概念，这等同于它的数据线 DQ 的数量。

假设 SoC 的 DRAM 位宽为 32bit ，对应的 DRAM 的位宽也要 32bit ，则：

  • 对于 DDR3/DDR4 ：假设它们的单颗粒位宽为 16bit ，Channel 也为 16bit ，则需要两个 DRAM 颗粒组成 32bit 。
  • 对于 LPDDR4/LPDDR4X ：假设它们的单颗粒位宽为 32bit ，但内部的 Channel 位宽为 16bit ，则单颗粒内部由 2 个 Channel 组成，见以下例图：

换句话说，DRAM颗粒的位宽取决于它内部的 Channel 。比如说 DDR3/DDR4 颗粒的位宽为 16bit ，它内部仅有一个 16 bit 的 Channel ；LPDDR4/LPDDR4X 颗粒的位宽为 32bit ，它内部有 2 个 16 bit 的 Channel ，组成 32 bit 位宽。

Rank 是内存的逻辑结构的概念，由多个 DRAM 颗粒组成一个 Rank ，共享同一组控制信号协同工作，通过 CS_n 片选信号选择不同 Rank 。举个简单的例子，比如说电脑主板的内存插槽，一个内存插槽对应一个 Rank 。

通过下列例子计算颗粒的容量，以此来辅助理解以上简述的概念。

以下截至 LPDDR4 的规格书：

从 Datasheet 抓取以下信息：

  • Row Address：R0-R15
  • Column Address：C0-C9
  • Number of Banks per Channel：8
  • Number of Channel per die：2
  • Bit ：16（等同于 DQ 位数）

换算过来，得出：

  • 行bit数 = 2^16 = 65536
  • 列bit数 = 2^10 = 1024
  • bank数 = 2^3 = 8
  • 位宽：16bit per channel * 2 CH = 32bit

则颗粒容量为：65536 * 1024 * 8 * 16 * 2 = 16Gb = 2GB 。

以下截至 DDR4 的规格书：

从 Datasheet 抓取以下信息：

  • Row Address：R0-R14
  • Column Address：C0-C9
  • Bit ：16
  • Number of Banks ：4
  • Number of Bank ：2
  • Number of Channel per die：1

换算过来，得出：

  • 行bit数 = 2^15 = 32768
  • 列bit数 = 2^10 = 1024
  • bank数 = 4*2 = 8
  • 位宽：16bit

则颗粒容量为：32768 * 1024 * 8 * 16 * 1 = 4Gb = 512MB 。

如果使用两片 512MB DDR4 ，则组成 1 Rank 的 1GB 内存。

参数调试原理

DRAM调试内容

  • 读写训练
    • DRAM的读写训练是确保内存子系统稳定运行的核心环节，DRAM IO信号读写眼图以及采样点的选择，决定了DRAM系统的性能和稳定性。
    • 时序参数：时序参数定义了DRAM操作的时间约束，DRAM调试需确保时序参数满足DRAM规格，并且均衡稳定性以及提高带宽、降低延迟之间的关系，此外还要匹配不同的DRAM颗粒等。
    • 信号Skew：在高频的应用下，各信号之间的skew会影响到其能否被接收端正常采样，skew的来源多种多样，有SOC控制器内部的链路、SOC基板打线和走线、PCB走线、颗粒基板及其Die内部链路、PVT等各种因素的影响。
  • 信号完整性调试
    • 信号完整性调试就是结合仿真和实测，对阻抗匹配、眼图质量、时序裕量和电源噪声等关键指标进行优化的过程
    • 比如调整驱动电阻、ODT电阻优化眼图，优化走线长度匹配来减小时序偏差、提升时序裕量
    • 比如优化电源层、参考层设计、调整合适的去耦电容来抑制电源纹波，提升高频的稳定性
    • 比如针对CA裕量窄的模板，优化CA信号端接，来解决其阻抗不匹配问题，提升CA裕量
    • 比如利用示波器捕获信号眼图，通过叠加波形的信号完整性进行问题的定位

什么是眼图

DRAM 使用差分信号传输数据，这时需要时钟信号为数据信号进行时序定位，采用双边沿 DDR 采集数据，见下图。

TCLKp 对应一个时钟周期，1UI 对应 0.5 个时钟周期，要求在上升沿或下降沿前，在 T_SETUP 期间准备好数据，且要在 T_HOLD 期间保持数据。

注意数据链路时序的阴影部分，这是信号转换区域（时域），它是眼图中重要的区域之一。这个区域由无数个的上升沿和下降沿信号叠加而成，形成交叉点（称之为 Jitter）。另外，这张例图还未展示电压轴，这是无数个高低电平信号叠加而成，形成“眼皮”（称之为 Noise）。

眼图由无数的信号叠加而来，在目前的行业标准中，以时钟为基准，使用 3bit 电平的波形叠加组合，以此形成眼图：

眼图中的信号抖动（Jitter）和噪声（Noise）的表现为：

  • 在横轴（时间轴）上，阴影区域的水平范围体现了时域抖动（Jitter）。假设眼高固定，眼宽越窄，即左右两边的阴影区域越宽，中间的空白区域越窄，则 T_SETUP 和 T_HOLD 时序越小，无法采样到数据。
  • 在纵轴（电压轴）上，阴影区域的垂直范围体现了电压噪声（Noise）。假设眼宽固定，眼高越矮，即上下两边的阴影区域越高，中间的空白区域越矮，则高低电平达不到接收端的 V_IHHS 和 V_ILHS（对应图 D-PHY_Signaling_and_Contention_Voltage_Levels ），无法采集到数据。

结合在一起理解，由于多次信号的叠加，如果：

  • 眼图的信息线变粗：出现模糊（Blur）现象，表示眼图越小，裕量越不充足，信号质量越差，接收端无法获取到数据。即阴影区域变宽，空白区域变窄。
  • 眼图的信息线变细：不会出现模糊（Blur）现象，表示眼图越大，裕量越充足，信号质量越好，接收端能很好地获取到数据。即阴影区域变窄，空白区域变宽。

在后续的读写训练中，涉及到一个名为“相位”的概念，这也是和数据线和时钟线之间的最佳采样点有关。和 Skew 不同的是，它用角度域表示。在理想状态下，数据线和时钟线之间的相位差为 180 度来作为最佳采样点，见下图：

多个相位（0°、90°、180°、270°、360°）只是一个标注拆分，并不表示采样点。

读写训练

要求

读写训练就是为了帮助每个信号找到眼图上的最佳采样点，这个最佳的点包含时间轴和电压轴两个方向。

电压轴：寻找时间裕量最大的电压值，为最佳Vref值：

时间轴：在最佳Vref点上找到左右边界，确定中心值为最佳采样点：

  • CK、CA、CS信号的相位：
    • 初始时的相位要求：CK的相位比CA/CS大90度，从而满足采样的时序要求
  • WDQS、WDQ信号的相位
    • wDQS信号与CK之间的相位，从PHY端发出是基本对齐的
    • 初始时的相位要求：DDR3/DDR4/LPDDR3要求wDQS的相位要比wDQ大90度；LPDDR4/4X要求wDQ的相位要比wDQS大90度
  • RDQS、RDQ信号的相位
    • rDQS信号与CK之间的相位关系由颗粒端保证。
    • 颗粒发出的rDQS和rDQ是边沿对齐的
    • 初始时的相位要求：主控端要将rDQS的相位设置得比rDQ大90度

数据IO端接模型

接下来到调试信号眼高范围，不同的DRAM类型，最优的电压轴的采样点位置不同，因为其端接电阻（ODT）的接法不同，导致其信号眼图的摆幅不同。

  • 输出驱动模型：输出高电平时上拉导通，输出低电平时下拉导通
  • 端接ODT模型：各类型的IO端接模型根据协议规定不尽相同

DDR3、DDR4、LPDDR3只有DQS、DQ和DM上有端接电阻，而LPDDR4和LPDDR4X除了DQS、DQ和DM之外，在CK和CA信号（除RESET和CKE）上也有端接电阻。

  • DDR3端接电阻是接到VDDQ/2，故其最优点在VDDQ/2左右；
  • DDR4和LPDDR3端接电阻是接到VDDQ，故其最优电压采样点在VDDQ到VDDQ/2之间；
  • LPDDR4和LPDDR4X端接电阻是接到VSSQ（即VDDQ的地），故其最优电压采样点在VDDQ/2到VSSQ之间。

具体的最优电压点采样数值与IO驱动电阻阻值和终端端接电阻阻值相关，需由实际调试来确定。

  • dram_type ：DRAM 类型配置

容量配置

  • dram_para1：配置bank_group、bank、row、col数目
  • dram_para2 ：配置DRAM使用空间、rank数量、位宽width

驱动ODT

  • dram_dx_odt：控制器数据DX读ODT阻抗
  • dram_dx_dri ：控制器数据DX输出驱动能力
  • dram_ca_dri ：控制器CK、CA输出驱动能力

Delay

  • dram_para0：调整DQS写方向的相位延迟
  • dram_tpr14：调整DQS读方向的相位延迟
  • dram_tpr11：调整DQ写方向的相位延迟
  • dram_tpr12：调整DQ写方向的相位延迟
  • dram_tpr0：调整CK、CA、CS信号的相位延迟

VREF

  • dram_tpr6：配置主控端DX读数据内部VREF
  • dram_mr6：for DDR4， [7:0]配置颗粒端DX写数据VrefDQ
  • dram_mr14：for LPDDR4， [5:0]配置颗粒端DX写数据VrefDQ

兼容性验证

验证标准

  • 高带宽读写压力测试：高低温下，不出现DRAM读写错误、memtester报错，系统不跑飞、不卡死，DRAM带宽满足要求，测试时长满足标准。
  • 反复初始化DRAM（Reboot重启）测试：高低温下，系统正常重启，不出现系统跑飞、卡死、硬件关机无法启动等现象，测试时长满足标准
  • DRAM自刷新（休眠唤醒）测试：高低温下，系统正常待机休眠和唤醒，不出现系统跑飞，卡死，硬件关机无法启动等现象，测试时长满足标准

报错修复

如果出现跑飞，卡死，则检查：

  • 排查是否裕量不足，参数是否还能优化
  • 降低DRAM工作频率来定位
  • 补焊颗粒是否有效
  • 确认是否与温度相关
    • 如与低温相关，可考虑给DRAM电源提压
    • 如与高温相关，是否刷新率不够等等
