安装

在内核目录的子目录 tool/perf ，里面有 perf 的源码，使用以下命令编译为可执行程序：

record

监测所有 CPU 的事件，并将其保存到文件“perf.data”中：

  • -F ：指定采样频率，一般为 49 或 99
  • -g ：记录栈追踪
  • -a ：监测所有 CPU
  • -v ：监测时显示详细信息，使用 -vv 可输出更详细。

指定 PID 或 TID 监测事件，并将其保存到文件“perf.data”中：

  • -p ：指定 PID 去监测该进程
  • -t ：指定 TID 去监测该线程
  • --call-graph ：使用指定的方法去追踪栈，默认使用 fp ，有些 Linux 发行版，如 Ubuntu ，需要指定 --call-graph dwarf 才能正确地追踪栈
    • fp ：使用基于栈指针的栈遍历技术
    • dwarf ：使用基于软件调试信息的栈遍历技术
    • lbr ：这是 Intel 的技术，最后分支记录（last bench record）栈遍历技术

参数 max-stack=n 用于给一个监测事件，指定追踪栈的最大深度。

report

用于总结 perf.data 的内容，常用的选项如下：

  • --tui ：使用 TUI 界面，可交互式界面。这是默认选用的。
  • --stdio ：以文本的方式输出报告
  • -i file ：选择文件总结
  • -n ：加入样本计数列
  • --header-only ：仅输出 perf.data 的头部信息

stat

该命令用于监测并统计事件，度量事件的频率，或监测一个事件是否发生了。
监测所有 CPU 的事件，或指定 PID 或 TID 监测事件：

  • -a ：监测所有 CPU
  • -p ：指定 PID 去监测该进程
  • -t ：指定 PID 去监测该线程

指定事件去监测并统计：

  • -e ：记录指定的事件（依赖于Ftrace），用,隔开。可以使用命令 perf list 查看可检测的事件，可以使用 perf list [type] 指定类型查看可检测的事件，如 perf list sched 。

指定事件去监测，并每隔一个周期打印一次统计信息（单位：ms）

  • -I interval_ms ：设置每隔 interval_ms 打印一次统计信息。

top

类似命令 top ，实时监测并显示系统级函数性能热点。以下是常用参数：

  • -a ：监测所有 CPU
  • -p ：指定 PID 去监测该进程
  • -t ：指定 PID 去监测该线程
  • -e ：记录指定的事件（依赖于Ftrace），用,隔开。可以使用命令 perf list 查看可检测的事件，可以使用 perf list [type] 指定类型查看可检测的事件，如 perf list sched 。
  • -g ：记录栈追踪
  • --call-graph ：使用指定的方法去追踪栈，默认使用 fp ，有些 Linux 发行版，如 Ubuntu ，需要指定 --call-graph dwarf 才能正确地追踪栈
