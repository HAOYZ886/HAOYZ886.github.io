I2C学习笔记

Posted on 2024-03-13 In Hardware Word count in article: 2k Reading time ≈ 7 mins.

i2c 是什么

i2c 是一条外设总线，是一条慢速 I/O 总线，供给慢速设备使用，如 EEPROM 。
芯片（SoC）内置 i2c 控制器，用于连接主板的其中一条 i2c 总线，且作为当前总线的主设备。一条 i2c 总线上有一个主设备和若干个从设备，总线上的主设备有主动发送请求的权限，从设备仅有回应请求的权限。

SoC 芯片能连接多少条 i2c 总线，取决于 i2c 控制器的数量。

其中，SCK 是时钟频率引脚，主从设备都可以控制这条引脚；SDA 是数据引脚，用于传输设备。

通信机制

先从逻辑层面上理解

当用户使用主设备，即在主控芯片发送请求时，需要提供目的设备和请求类型。
i2c 总线上的从设备由一个 7 bits 的设备地址表示。在一条 i2c 总线，设备地址是唯一的，一个 i2c 设备对应一个设备地址，一个 i2c 设备只能连接一条总线。而且不同的 i2c 总线，相同地址所对应的设备是不同的。
请求类型分为读和写，用 1 bit 表示，0 表示 写，1 表示读。
把信息提供给 i2c 驱动层后，i2c 驱动层（里面的传输层）会先通过 SDA 引脚发送 start 信号（被置为 0 ），然后逐位发送设备地址 + 请求类型到 i2c 从设备。当总线上的其中一个从设备匹配时，它通过 SDA 引脚会发送 ack 信号（被置为 0 ）给主设备，根据请求类型进行传输数据。在传输期间，传输 1 字节为一个周期，每一周期，接收方必须发送 ack 信号给发送方。当传输完成时，主设备会发送 stop 信号给从设备，表示传输结束。所以可以得知，i2c 提供了一定的可靠性。

个人认为，在主设备发送 start 位时，会通过广播的方式，逐位发送 start + dev_addr + rw 到当前总线的各个从设备，当这个信息发送到与 dev_addr 相符的从设备后，该从设备必须发送 ack 信号给主设备，然后再根据 rw 传输数据。

i2c 写操作

  1. 主芯片发出一个 start 信号
  2. 然后发出一个设备地址（确定是往哪一个从设备写数据），方向（写操作，0 ）
  3. 从设备回应（用来确定这个设备是否存在），然后就可以传输数据
  4. 主设备发送一个字节数据给从设备，并等待回应
  5. 每传输一字节数据，从设备发送一个 ack 信号（确定数据是否接受完成），然后再传输下一个数据。
  6. 数据发送完之后，主芯片就会发送一个停止信号。

i2c 读操作

  1. 主芯片发出一个 start 信号
  2. 然后发出一个设备地址（确定是往哪一个从设备读数据），方向（读操作，1）
  3. 从设备回应（确定这个设备是否存在），然后就可以传输数据
  4. 从设备发送一个字节数据给主芯片，并等待回应
  5. 每传输一字节数据，主芯片要有一个回应信号（确定数据是否接受完成），然后再传输下一个数据。
  6. 数据发送完之后，主芯片就会发送一个停止信号。

这个阶段需要知道 SDA 和 SCL 之间的关系。

当主设备发送 start 信号前，SDA 和 SCL 都是高位（与上拉电阻有关，后面会解释）。发送之后，SDA 变低位，随后 SCL 也会变低位。在 SCL 低位期间，表示主设备可以发送数据到 i2c 总线，此时会发送设备地址的一位到总线（从高到低的位数次序）。在 SCL 变高位期间，SDA 的位需要保持，供给从设备接收。待一个时钟周期结束后，即 SCL 变低位后，主设备才能发送下一位到总线。总体流程如下图：

这里或许会有一些疑问：
Q: SCL 是谁主导的？
A: 由上文可知，当主设备发送 start 信号时，SCL 就由主设备控制了。但是，当传输完 dev_addr + rw 后，dev_addr对应的从设备必须发送 ack 信号，所以轮到当前从设备控制。

Q: 如何在 SDA 上实现双向传输？
A: 主芯片通过一根 SDA 线既可以把数据发给从设备，也可以从 SDA 上读取数据，连接 SDA 线的引脚里面必然有两个引脚（发送引脚/接收引脚）。

Q: 双方设备中，某个设备发送数据时，另一方怎样才能不影响 SDA 上的数据？
A: 这个取决于 i2c 的硬件机制。已知 SDA 和 SCL 分别外接了一个 3.3v 的上拉电阻，这使得 SDA 和 SCL 在没有被主从端传输时都处于高位。而且，在 i2c 控制器的 SDA 中有一个三极管，使用开极/开漏电路。见下图：

  • 当驱动三极管时，SDA 处于低电平。
  • 不驱动三级管时，SDA 处于什么电平，取决于上拉电阻和发送端。

举例：主设备发送（ 8bit ）给从设备

  • 前 8 个 CLK
    1. 从设备不影响 SDA ，从设备不驱动三极管。
    2. 主设备决定数据，主设备要发送 1 时不驱动三极管，要发送 0 时驱动三极管.
  • 第 9 个 CLK，由从设备决定数据。
    1. 主设备不驱动三极管。
    2. 从设备决定数据，要发出回应信号的话，就驱动三极管让 SDA 变为 0 从这里也可以知道 ACK 信号是低电平。

由此发现一个巧妙的设计，发送高电平始终都是由上拉电阻支持，发送低电平仅由各端的三极管决定。避免了各端发送不同的电平时导致线路短路。

为何 SCL 也要使用上拉电阻？在第 9 个时钟之后，如果有某一方需要更多的时间来处理数据，它可以一直驱动三极管把 SCL 拉低。
当 SCL 为低电平时候，大家都不应该使用 i2c 总线，只有当 SCL 从低电平变为高电平的时候，i2c 总线才能被使用。
当它就绪后，就可以不再驱动三极管，这是上拉电阻把 SCL 变为高电平，其他设备就可以继续使用 I2C 总线了。
