Linux - 调试i2c技巧

Posted on 2024-06-23 In Skill Word count in article: 1.2k Reading time ≈ 4 mins.

查看

列出当前的 I2C Adapter(或称为 I2C Bus、 I2C Controller)

打印某个 I2C Adapter 的 Functionalities, I2CBUS 为 0、 1、 2 等整数
i2cdetect -F I2CBUS

看看有哪些 I2C 设备, I2CBUS 为 0、 1、 2 等整数
i2cdetect -y -a I2CBUS

--表示没有该地址对应的设备, UU 表示有该设备并且它已经有驱动程序, 数值表示有该设备但是没有对应的设备驱动。

I2C 读

使用示例：

读一个字节: I2CBUS 为 0、 1、 2 等整数, 表示 I2C Bus; CHIP-ADDRESS 表示设备地址

读某个地址上的一个字节:

  • I2CBUS 为 0、 1、 2 等整数, 表示 I2C Bus
  • CHIP-ADDRESS 表示设备地址
  • DATA-ADDRESS: 芯片上寄存器地址
  • MODE：有 2 个取值, b-使用SMBus Read Byte先发出 DATA-ADDRESS, 再读一个字节, 中间无 P 信号。c-先 write byte, 在 read byte，中间有 P 信号
  • 读某个地址上的 2 个字节:
  • I2CBUS 为 0、 1、 2 等整数, 表示 I2C Bus
  • CHIP-ADDRESS 表示设备地址
  • DATA-ADDRESS: 芯片上寄存器地址
  • MODE： w-表示先发出 DATA-ADDRESS，再读 2 个字节 i2cget -f -y I2CBUS CHIP-ADDRESS DATA-ADDRESS MODE

I2C 写

使用示例：

写一个字节: I2CBUS 为 0、 1、 2 等整数, 表示 I2C Bus; CHIP-ADDRESS 表示设备地址 DATA-ADDRESS 就是要写的数据

给 address 写 1 个字节(address, value):

  • I2CBUS 为 0、 1、 2 等整数, 表示 I2C Bus; CHIP-ADDRESS 表示设备地址
  • DATA-ADDRESS: 8 位芯片寄存器地址;
  • VALUE: 8 位数值
  • MODE: 可以省略，也可以写为 b

给 address 写 2 个字节(address, value):

  • I2CBUS 为 0、 1、 2 等整数, 表示 I2C Bus; CHIP-ADDRESS 表示设备地址
  • DATA-ADDRESS: 8 位芯片寄存器地址;
  • VALUE: 16 位数值
  • MODE: w

SMBus Block Write：给 address 写 N 个字节的数据
发送的数据有： address, N, value1, value2, …, valueN 。跟I2C Block Write相比, 需要发送长度 N

  • I2CBUS 为 0、 1、 2 等整数, 表示 I2C Bus; CHIP-ADDRESS 表示设备地址
  • DATA-ADDRESS: 8 位芯片寄存器地址;
  • VALUE1~N: N 个 8 位数值
  • MODE: s

I2C Block Write：给 address 写 N 个字节的数据
发送的数据有： address, value1, value2, …, valueN 。跟SMBus Block Write相比, 不需要发送长度 N

  • I2CBUS 为 0、 1、 2 等整数, 表示 I2C Bus; CHIP-ADDRESS 表示设备地址
  • DATA-ADDRESS: 8 位芯片寄存器地址;
  • VALUE1~N: N 个 8 位数值
  • MODE: i
